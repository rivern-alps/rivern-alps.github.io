python对音频文件处理

简介

由于quecpython usr目录下放置文件时，是按照block来，一个block为4K,当文件过多时，会造成资源浪费。而且当文件过多过大时，下载时间长且容易造成文件下载失败，影响客户实际功能。故定义该方案，多文件时，按照一定规则将该文件合并为一个文件，以节省资源。

背景

为配合模组端音频文件合并拼接的方案，需要一个上位机工具，可以自动化实现音频文件的生成、内容裁剪、拼接，最后拼接后的文件可以通过模组端的接口来播放。

拼接方案

在上位机上按照如下设定格式，对文件进行拼接。格式如下：

标识（8b） 文件个数（4b） 第一个文件名（128b）
第一个文件偏移量(4b) 第一个文件大小（4b） …… 第n个文件名（128b）
第n个文件偏移量(4b) 第n个文件大小（4b） 第一个文件完整数据（大小取决于第一个文件的大小）
……
第n个文件完整数据（大小取决于第n个文件的大小）
数据 大小 备注
文件识别标识 8byte(固定”montage”) 用于识别该文件为拼接文件
包含的文件数 4byte 该拼接文件是由多少个文件拼接的
第一个文件的数据信息 128byte + 4byte + 4byte (文件名 + 偏移量 + 文件大小) 文件名128byte,不足补0 偏移量：是基于整个文件的偏移 文件大小：该文件的长度
… … …
第n个文件的数据信息 128byte + 4byte + 4byte (文件名 + 偏移量 + 文件大小) 文件名128byte,不足补0 偏移量：是基于整个文件的偏移 文件大小：该文件的长度
第一个文件的完整数据 第一个文件大小
… … …
第n个文件的完整数据 第n个文件大小

方案实现

以下实现方案均为python实现

实现音频拼接

音频文件生成

可以通过百度开放平台或讯飞等python SDK实现

SDK使用说明

生成后的音频文件会存在空白的结尾，在播放时会 产生停顿感，所以需要将音频文件尾部空白部分裁剪后再进行拼接，同时由于生成的文件为wav无损格式，在裁剪后需要将文件转换成MP3或amr格式后再进行拼接以减小占用空间。

实测转换成amr后一个字的语音文件占用空间在900字节左右

音频文件裁剪

步骤

  1. 通过python wave库对上一部生成的wav文件进行读取，获取音频的通道、量化位数、采样频率、采样点数
  2. 读取采样点数数据获取波形数据
  3. 通过numpy将波形数据转换成np数组
  4. 对数组数据进行分析将尾部的空数据剔除
  5. 通过采样频率和采样点数获取音频时间，再根据上一步获取的剔除数据位置截取有效的音频段（裁剪音频文件需要pydub库，读取wav数据后可以根据时间戳来剪切指定数据）

音频文件转换

通过win上的ffmpeg工具进行音频格式转换

注意：win上使用ffmpeg工具需要配置环境变量并且重启（不重启的话python无法获取到环境变量）

转换之后使用第一步的拼接步骤即可获得最终的音频包，传入模组中调用播放接口即可播放
