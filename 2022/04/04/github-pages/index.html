在搭建好hexo之后下一步就是考虑如何将内容部署到服务器上，如标题所言，这里用的是Github Pages！

hexo自动部署

Travis CI：travis-ci.org 专门针对开源项目，github私有库免费

Githooks： 这个如果 vps 本地部署，配合 nginx ，还是很推荐的

Github Actions： 持续集成，自动执行软件开发工作流程

GitHub Actions

准备项

这里我们用的是github actions，首先需要使用hexo的deploy来部署，需要安装插件

然后在项目的根目录配置_config.yaml内容

环境准备

  1. 私有仓库 存放hexo项目文件
  2. 公共仓库 要求名称：用户名.github.io，用来 发布静态页面的，要求空仓库，没有初始化及任何操作的

秘钥准备

GitHub Actions运行时需要登录GitHub账号，所以设备的私钥交给GitHub Actions，公钥交给GitHub

具体操作

首先使用ssh-keygen生成一组公私秘钥对，输入以下命令，一直回车就会生成两个文件，其中pub后缀的是公钥，另外一个是私钥

  • 配置公钥，配置路径：github网站–>Settings–>SSH and GPG keys
  • 配置私钥，blog私有仓库的Settings->Secrets里添加私钥，名称为HEXO_DEPLOY_PRIVATE_KEY

配置GitHub Actions

在blog仓库的Actions选项卡下点击新建workflow，名称默认或者自定义修改，编写如下配置，也可以在

仓库目录\.github\workflows\deploy.yml中编辑（没有就创建），提交代码后会触发自动流程，在仓库的actions中可以看到结果和日志（也会有邮件通知）
